# Release & Rollout Plan — platform-router v1.0.0

- [x] All tests pass (46/46)

- [x] `npm audit` — 0 vulnerabilities
- [x] Dockerfile builds
- [x] Helm chart complete
- [x] README complete
- [ ] Integration tests (requires Docker + Testcontainers)
- [ ] Load tests (requires k6)

## Release Steps

### 1. Tag
```bash
git tag -a v1.0.0 -m "platform-router v1.0.0 — initial release"
git push origin v1.0.0
```

### 2. Build & Push Docker Image
```bash
docker build -t ghcr.io/yourorg/platform-router:1.0.0 .
docker push ghcr.io/yourorg/platform-router:1.0.0
```

### 3. Deploy Canary (1 replica)
```bash
helm install platform-router ./helm \
  --set replicaCount=1 \
  --set image.tag=1.0.0 \
  --namespace platform-router
```

### 4. Validate Canary
- Check `/healthz` returns 200
- Check `/metrics` shows `router_processed_total` incrementing
- Check Kafka consumer group `platform-router` is active
- Verify `routed_job` events on `platform-router.routed_jobs` topic
- Monitor `router_errors_total` — should be 0

### 5. Rollout to Full (3 replicas)
```bash
helm upgrade platform-router ./helm \
  --set replicaCount=3 \
  --set image.tag=1.0.0 \
  --namespace platform-router
```

### 6. Monitor
- Prometheus: `router_latency_seconds{stage="routing"}` p95 < 10ms
- Prometheus: `router_errors_total` rate < 0.01/s
- Kafka: consumer lag stable
- K8s: all pods ready, no restarts

## Rollback

```bash
helm rollback platform-router --namespace platform-router
```

Or pin previous image tag:
```bash
helm upgrade platform-router ./helm --set image.tag=0.9.0
```

## Post-release

- Update `config/tasks.json` all milestones to `"complete"`
- Archive this file to `docs/releases/v1.0.0.md`
